# GENESIS demo script (under three minutes)

## 0:00-0:20 — The problem

“AI coding tools make production incredibly fast. But speed also makes it easy to build the wrong product, in the wrong order. GENESIS is the product architect that sits before production.”

Show the landing view and its seven-stage rail.

## 0:20-0:42 — Begin with a vague idea

Select **Use sample idea** and read the input:

“Build a service that helps secondary-school teachers identify struggling students before examinations.”

Select **Shape the blueprint**.

## 0:42-1:05 — Distributed specialist intelligence

While the progress view is visible:

“GPT-5.6 Sol examines the idea through vision, user, risk and systems perspectives. A central synthesis resolves those findings into one coherent blueprint.”

## 1:05-1:42 — Purpose, boundaries and premortem

Show:

- the primary user and observable outcome;
- essential features beside the explicit non-goals;
- the specialist perspectives;
- one high-impact premortem risk and its early validation test.

Say: “GENESIS marks unsupported claims as assumptions. It does not invent market evidence.”

## 1:42-2:17 — Foundations and creation order

Scroll through foundations and build phases.

“The most visible feature does not necessarily come first. GENESIS asks what must already exist, then creates in dependency order.”

Show the Codex task pack: expected files, acceptance criteria, tests, prohibited scope and definition of done.

## 2:17-2:40 — Complete and export

Show the completion review, then select **Download Markdown**.

“The complete plan is now a portable, repository-ready build pack for Codex.”

## 2:40-2:58 — OpenAI and Codex contribution

“GENESIS uses the OpenAI Responses API with GPT-5.6 Sol and schema-validated Structured Outputs. Codex built and verified the primary implementation workflow, from typed architecture through responsive browser tests.”

## Close

“GENESIS does not merely help you code faster. It helps you create wisely.”
